  GIANNI Custom v2.0 — POPRAVKE (uplod paket)
═══════════════════════════════════════════════════════════════

✅ 1) KALADONT — riječi se sad PROVJERAVAJU u rječniku
   • Dodan veliki BHS rječnik (preko 700 riječi: hrana, životinje,
     priroda, profesije, sport, geografija, glagoli, pridjevi…).
   • "KT", "QU" i sl. gluposti se ODBIJAJU sa porukom.
   • Riječ se normalizuje (Š→S, Č→C, Ž→Z, Đ→DJ, Ć→C) pa nema
     problema sa dijakritikom.
   • Magična riječ "KALADONT" i dalje radi kao pobjeda.

✅ 2) .HELP prikazuje "." prefix kad se pozove sa "."
   • Detektuje da li je pozvano kao slash (/) ili prefix (.).
   • Ako je .help → embed je u "MOBITEL MODU": kompaktniji,
     manje sekcija, sve komande pišu sa "." umjesto "/".

✅ 3) VATRICE +1 svakih 150 poruka — robusno (radi i nakon uploada)
   • Umjesto "msg_count % 150 == 0" (koje gubi vatrice ako se broj
     poruka prebroji ili preskoči), sada koristi THRESHOLD logiku:
     prati `data["vatrice_threshold"][user]` i dodjeljuje sve
     vatrice koje su zaslužene od zadnjeg threshold-a.
   • Nakon uploada/restarta: bot automatski podesi threshold
     na trenutni najbliži milestone (npr. ako član ima 470
     poruka → threshold 450, sljedeća vatrica na 600).

✅ 4) PERZISTENCIJA tickets/staff/giveaways
   • Već je bilo dobro postavljeno (TicketOpenView, TicketCloseView,
     StaffVoteView, GiveawayView svi imaju `custom_id` i registruju se
     preko `bot.add_view()` u on_ready).
   • Aktivni giveawayi se restoruju iz `data["giveaways"]` u on_ready.
   • Brojanje, vatrice, ekonomija itd. se spašavaju u oleun_data.json
     atomski (temp + rename + .bak).
   • Cloud backup se uploaduje na Discord kanal (BACKUP_CHANNEL_ID)
     i automatski restoruje pri restartu — preporučeno postaviti
     env var BACKUP_CHANNEL_ID za 100% sigurnost.

✅ 5) AKTIVNOST u VATRICE EMBER
   • `/vatrice ember @član` sada prikazuje:
       💬 Napisanih poruka (ukupno)
       📅 Poruka ove sedmice
       🔥 Vatrice ukupno
       🎯 Do sljedeće auto-vatrice (X poruka)

✅ 6) BROJANJE — ne resetuje se nakon uploada
   • `data["counting"]` se već spašava u oleun_data.json i učitava
     pri restartu (kroz fallback `for k,v in loaded.items()` u
     load_data()), ali sada je i eksplicitno dodano u rječnik
     `vatrice_threshold` slot.
   • Ako KORISTITE Discord cloud backup — postavi env var
     BACKUP_CHANNEL_ID da bot automatski povuče backup pri restartu.

✅ 7) .DAILY cooldown 24h → 1h
   • `/daily` sada radi svakih **1 sat** umjesto svakih 24h.
   • Embed i opis komande ažurirani (Hourly nagrada).

  KAKO INSTALIRATI
═══════════════════════════════════════════════════════════════
1) Zamijeni stari bot.py sa ovim novim.
2) `pip install -r requirements.txt` (ako prvi put).
3) Postavi env var DISCORD_TOKEN.
4) (opciono) BACKUP_CHANNEL_ID = ID tvog backup kanala
   (Bot mora imati Send + Attach Files pristup tu).
5) Pokreni: `python3 bot.py`
